# Ecolingo — Execution Summary

**First course:** ECON 13210 — Introduction to Macroeconomic Models (proving ground, not a permanent limitation).
**Core loop:** teacher materials → concept map → personalized lessons → adaptive practice → mastery data → better next lesson.

This document is the programme's living index: scope decisions, the decision log, the risk register pointer, the source-provenance map, and the mapping of the 29 required output artifacts to files and owners.

## 4. Technical stack (as specified §18, confirmed)

Next.js + React + TypeScript + Tailwind; KaTeX for math (MathML output for screen readers); code-rendered SVG for all truth-critical graphs (JSXGraph optional later); Supabase Postgres/Auth/Storage/pgvector with RLS (Phase 1); Trigger.dev-class job runner for ingestion (Phase 2); Vercel AI SDK-style provider abstraction; PostHog-class analytics; Sentry, Storybook, Playwright, Vitest. Rive is reserved for mascot/feedback states **after** Fabel delivers the design system (never for truth-critical diagrams).

## 5. Division of work between agents (§19)

| Agent | Owns |
|---|---|
| Long-horizon agent (this session's role) | Architecture, PRD, curriculum compiler design, schemas, consistency review, coordination |
| Coding agent(s) | Feature implementation, migrations, components, tests, security review (Phases 1+) |
| **Fabel** | Brand, visual identity, mascot, motion, landing-page and screen visual design (per CLAUDE.md) |
| Higgsfield MCP | Marketing/hero video, mascot exploration, chapter transitions — never examinable-model visuals |
| Fast image model | Illustrations/thumbnails — never equations, graphs, answer keys (GATE-002) |

Model-tiering rule: strongest models for offline compilation and complex teacher requests; fast models for classification/hints/ordinary explanations; cache repeated grounded explanations.

## 7. First shippable experience — the bar

> A struggling learner can understand a difficult model through a personalized explanation and a precise interactive visual, demonstrate transfer, and be reviewed again before forgetting.

The slice in `app/` walks that path end-to-end for the Solow steady state: adaptive lesson steps → Solow Lab manipulation → transfer question → misconception-specific feedback → mastery update → explainable scheduled review.
